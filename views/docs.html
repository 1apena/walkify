Getting Started

to use walkify, you must have a parent container to hold your views. e.g
<div class="app" id="app"> </div>
the next step is to instantiate a routes Object
let routes = { } //instatiating router with routes Object as the argument; let router = new Walkify(routes); //mounting to the container with class 'app' router.mount('.app'); /* * or router.mount('#app'); */ /* * or one liner * let router = new Walkify(routes).mount('.app'); */

walkify Routes Object

walkify Routes Object are where you will register your routes. you have to specify the routes e.g to register a default route ie index/landing view for your app

first of all create a template for your route. note the attribute 'for'. (not neccessary if you fetches the template with ajax)
<template for="index"> ${{ message }} </template>
then add the route to routes Object
let routes = { '/' : { //the value of the atrtribute 'for' must be the name name : 'index', /* matched hook /* walkify calls this method when it matches a registered route */ matched(){ let data = { message : "hello word" }; //render the view this.view(data); }, /* * exist hook * walkify calls this method before switching to another route */ exist(){ //show/hide animations and other stuffs } }, }
an example using an ajax fetched template
firstly create a folder named views in you projects root, then inside it create a file 'index.html'. then add the below text to nav-item
${{ message }}
then add below code in your route object
let routes = { '/' : { //the value of the atrtribute 'for' must be the name name : 'index', /* matched hook * walkify calls this method when it matches a registered route */ matched(){ let data = { message : "hello word" }; fetch('views/index.html') .then((res) => { //throwing an error on purpose, //uncomment below line to see how it works // throw new Error(''); return res.text(); }).then((template) => { hideLoader(); //inline template is ignored when you pass second argument this.view(data, template); }).catch((e) => { let data = { message : "something went wrong" } /* * redirecting to another route (404 route) with a message * note : this doesnt change the url, it just calls the specified route; * to force full redirect, use router.routeTo(route); * e.g router.routeTo('/about'); */ router.redirectTo('!', data); }); }, }, }

walkify Hooks

walkify provides three hooks/methods for route object
  • matched
  • mounted
  • exist

-matched

walkify always expects this method, it calls it when there is a match in your registered routes. it also accepts parameters when you register dynamic routes e.g
'/home/{id:[0-9]+}/test/{name:[a-y]+}' : { name : 'homeplus', matched(id, name){ let data = { id, name }; this.view(data); }, },

-mounted

walkify looks and call this method if it is available after the view has being rendered. at this part the dom is ready to be manipulated.
'/home/{id:[0-9]+}/test/{name:[a-y]+}' : { name : 'homeplus', matched(id, name){ let data = { id, name }; this.view(data); }, mounted(){ //Dom is Ready! //document.getElementById('example').innerHtml = 'test' } },

-exist

walkify looks and call this method if it is available on route change before handling the next route
'/home/{id:[0-9]+}/test/{name:[a-y]+}' : { name : 'homeplus', matched(id, name){ let data = { id, name }; this.view(data); }, mounted(){ //Dom is Ready! //document.getElementById('example').innerHtml = 'test' }, exist(){ //this view is being removed } },

walkify 404

to register a route for 404 pages, create a template to hold the view e.g
<template for="404"> ${{ errorMessage }} </template>
then register it in your routes object as follows
let routes = { '!' : { name : '404', matched(){ //this object (data) is ignored when this view is called from redirectTo method let data = { errorMessage : 'page not found' }; this.view(data); }, } } }
